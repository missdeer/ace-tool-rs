# MCP 搜索评测计划

## 目标

对当前会话已接入的 `ace-tool-rs` MCP 搜索能力做一轮真实评测，判断：

- 能否稳定返回结果
- 结果是否命中真正想找的内容
- 结果的噪声、聚焦度、可直接消费性如何
- 是否适合日常代码定位

## 评分口径

- 准确性：35 分
- 召回完整度：25 分
- 聚焦度：20 分
- 可直接消费性：20 分
- 总分：100 分

## 评测步骤

1. 梳理仓库中适合做检索基线的真实问题
2. 选出 5 个不同类型的问题，覆盖入口定位、调用链、配置、协议、测试
3. 对每个问题实际调用 `mcp__ace-tool__search_context`
4. 记录返回结果、命中文件、偏差和噪声
5. 用源码核验结果是否真的回答了问题
6. 汇总单题评分与总体结论

## 当前状态

- [completed] 设计评测题
- [completed] 执行 5 组搜索
- [completed] 逐题核验与评分
- [completed] 汇总结论和建议

## 已知限制

- 当前评测的是“本会话中已接入的 MCP 能力”，不等于独立新起一个 `ace-tool-rs` 进程的端到端安装验证
- 结果质量受索引状态、查询写法、后端召回策略共同影响
